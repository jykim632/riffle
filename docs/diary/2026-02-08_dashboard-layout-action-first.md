## 왜 했는지

멤버의 대시보드 방문 행동 패턴 분석:
1. "나 제출했나?" → WeekOverview (CTA)
2. "다른 사람은 뭐 썼지?" → CurrentWeekSummaries
3. "시장 어땠지?" → MarketSummaryWidget (부가)
4. "시즌 얼마나 남았지?" → SeasonBanner (부가)

기존 레이아웃은 핵심 행동(제출 → 읽기)이 스크롤 아래에 묻혀 있었음. 시즌 배너와 시장 요약이 상단을 차지하고, 요약본은 2칸 그리드 오른쪽(모바일에선 WeekOverview 아래)으로 밀려 있었음.

## 논의/고민

### 3가지 배치안 검토

| 안 | 구조 | 판단 |
|----|------|------|
| A (채택) | CTA → 요약본 → 시장 → 시즌 | 행동 흐름 일치, 모바일 최적 |
| B | 기존 유지 (순서만 미세 조정) | 변화 없음, 문제 해결 안 됨 |
| C | 시즌 → CTA(컴팩트) → [요약본 + 시장 2칸] | 중간안이지만 복잡도 증가 |

### WeekOverview 풀 와이드 우려
- max-w-7xl 안에서 카드가 너무 넓어 보일 수 있음
- 단, 내부가 space-y-8 세로 레이아웃이라 가로 여백이 자연스럽게 생김
- 추후 가로 레이아웃으로 변경 가능 (주차 정보 좌측 + 제출 현황 우측)

## 결정된 내용

- **배치 원칙**: 행동 빈도 순 (매주 반복 행동 > 참고 정보)
- **그리드 제거**: 풀 와이드가 모바일에서 일관적이고 스크롤 흐름이 자연스러움
- **SeasonBanner 하단**: 매일 볼 필요 없는 정보, 하단이 적절

## 난이도/발견

- 난이도: 낮음 (JSX 순서 변경만)
- "어디에 뭘 놓을 것인가"가 코드량과 무관하게 UX에 큰 영향을 줌
- 2칸 그리드는 콘텐츠가 비대칭일 때(카드 높이 차이) 오히려 불균형해 보임
