# 모바일 앱 전환 검토

> 작성일: 2025-02-08
> 현재 스택: Next.js 16 + Supabase + Vercel
> 서비스 특성: 소규모 폐쇄형 스터디 웹앱

## 배경

현재 웹 애플리케이션을 모바일 앱으로 전환할 때의 선택지, 비용, 트레이드오프를 정리한다.

---

## 옵션 비교

### 1. PWA (Progressive Web App)

| 항목 | 내용 |
|------|------|
| 추가 비용 | 0원 |
| 개발 기간 | 1~3일 |
| 작업 내용 | manifest.json + service worker 추가 |
| 스토어 등록 | 불가 |

**장점**
- 기존 코드 변경 최소
- 홈화면 추가로 앱처럼 사용 가능
- 오프라인 캐싱 지원
- 배포/업데이트가 웹과 동일 (즉시 반영)

**단점**
- iOS 푸시 알림 제한적 (Safari 17.4+부터 지원하나 불안정)
- 앱스토어/플레이스토어 등록 불가
- 네이티브 기능 접근 제한 (카메라, 파일시스템 등)

**적합한 경우**: 소규모 사용자, 스토어 등록 불필요, 빠른 적용 원할 때

---

### 2. Capacitor (웹 → 네이티브 래핑)

| 항목 | 내용 |
|------|------|
| 추가 비용 | Apple 개발자 $99/년 + Google Play $25 (1회) |
| 개발 기간 | 1~2주 |
| 작업 내용 | 기존 웹앱을 네이티브 WebView 쉘로 감싸기 |
| 스토어 등록 | 가능 |

**장점**
- 기존 코드 90% 이상 재사용
- 네이티브 플러그인으로 푸시 알림, 카메라 등 접근 가능
- 앱스토어/플레이스토어 등록 가능
- Ionic 팀이 관리하는 안정적인 프로젝트

**단점**
- 기본적으로 WebView라 네이티브 앱 대비 성능/UX 차이 존재
- 앱 심사 과정 필요 (특히 Apple)
- 업데이트 시 스토어 재심사 필요 (긴급 수정 지연 가능)

**적합한 경우**: 스토어 등록이 필요하지만 코드 재작성은 피하고 싶을 때

---

### 3. React Native / Expo

| 항목 | 내용 |
|------|------|
| 추가 비용 | 스토어 등록비 동일 |
| 개발 기간 | 3~6주 (1인 기준) |
| 작업 내용 | UI를 React Native 컴포넌트로 전면 재작성 |
| 스토어 등록 | 가능 |

**장점**
- 네이티브에 가까운 성능과 UX
- Supabase JS 클라이언트 그대로 사용 가능
- React 경험 활용 가능 (학습 곡선 낮음)
- Expo로 빌드/배포 자동화 가능

**단점**
- UI 코드 전면 재작성 필요 (HTML/CSS → RN 컴포넌트)
- 웹과 앱 두 코드베이스 유지 부담
- 마크다운 에디터 등 웹 전용 라이브러리 대체 필요

**재사용 가능한 부분**
- Supabase 클라이언트 설정
- Zod 스키마
- 비즈니스 로직 (유틸 함수 등)

**재작성 필요한 부분**
- 모든 UI 컴포넌트
- 라우팅 (React Navigation으로 전환)
- 마크다운 에디터 (react-native-markdown 계열로 교체)

---

### 4. 네이티브 (Swift / Kotlin)

| 항목 | 내용 |
|------|------|
| 추가 비용 | 외주 시 500만~2,000만원 |
| 개발 기간 | 1~3개월 |
| 작업 내용 | 전체 새로 개발 |
| 스토어 등록 | 가능 |

**장점**
- 최고의 성능과 네이티브 UX
- 플랫폼별 최적화 가능

**단점**
- 새 기술 스택 학습 또는 외주 필요
- iOS/Android 각각 개발 → 2배 유지보수
- 이 규모 앱에는 과도한 투자

**적합한 경우**: 대규모 사용자, 고성능 필요, 전문 개발팀 보유 시

## 비용 요약

| 옵션 | 초기 비용 | 유지 비용(/년) | 개발 기간 | 코드 재사용 |
|------|-----------|---------------|-----------|------------|
| PWA | 0원 | 0원 | 1~3일 | 99% |
| Capacitor | ~15만원 | ~12만원 | 1~2주 | 90%+ |
| React Native | ~15만원 | ~12만원 | 3~6주 | 30~40% |
| 네이티브 | 500만원+ | ~12만원 | 1~3개월 | 0% |

---

## 추천 전략: 단계적 접근

현재 프로젝트 상황(1인 개발, 소규모 스터디, 저트래픽)을 고려하면:

### Phase 1: PWA 적용 (즉시)
- manifest.json, service worker 추가
- 모바일 최적화 확인 (반응형 레이아웃)
- 비용 0원, 1~3일 소요

### Phase 2: 필요 시 Capacitor 래핑 (선택)
- 푸시 알림이 반드시 필요하거나 스토어 등록 요구 시
- PWA 코드 기반으로 래핑만 하면 됨
- Phase 1 완료 후 판단

### Phase 3: React Native 전환 (미래)
- 사용자 규모가 커지고 네이티브 UX가 필요해질 때
- 현실적으로 이 프로젝트에서는 필요 없을 가능성 높음

---

## 결론

**PWA가 최선.** 스토어 등록이 필요 없는 소규모 폐쇄형 서비스에서 네이티브 앱은 비용 대비 효과가 낮다. PWA로 시작하고, 부족한 점이 실제로 발생하면 그때 Capacitor로 확장하는 것이 1인 개발에서 가장 합리적인 전략이다.
